## 二、为什么不写App/产品对比？

根据已有研究：
- 用户明确不想写 app/产品对比（付费工具评测）
- 竞争报告已显示 StockBrokers.com 等网站在该领域占据优势
- 开源GitHub项目和策略对比是差异化机会，且与vibetrading.fun的"AI Trading Lab"定位更契合
- Reddit和GitHub数据已提供大量可直接转化的内容素材

## 五、每篇内容SOP标准

### Frontmatter模板
```yaml
---
title: "Your Post Title"
description: "A short description for SEO and cards."
date: "2026-07-17"
category: "Guides" # Guides | Reviews | Tutorials | Challenges | Tools | Strategies
tags: ["ai trading", "open source", "github"]
readTime: "10 min read"
featured: false
lang: "en"
faq:
  - question: "Question 1?"
    answer: "Answer 1."
  - question: "Question 2?"
    answer: "Answer 2."
---
```

### 内容质量标准
- [ ] 英文≥800字，教程/对比≥1500字
- [ ] 主关键词出现在标题、首段、至少1个H2、Meta description
- [ ] 至少3个站内内链（指向已有相关文章）
- [ ] 至少1张图片/表格/代码块
- [ ] 包含 `<Disclaimer />`
- [ ] 包含 FAQ frontmatter（自动生成FAQPage Schema）
- [ ] 中文版本不是简单翻译，需本地化（案例、术语、监管提示）
- [ ] 文末相关阅读推荐

### SEO检查清单
- [ ] URL slug 与英文标题关键词一致
- [ ] Meta description 150-160字符
- [ ] 图片含 alt text
- [ ] 内链锚文本多样化
- [ ] 代码块使用 ```python 等语法高亮

## 六、内容生产流水线

### 阶段1：批量生成英文MDX（Day 1-13）
1. 按日历顺序选择当日主题
2. 基于research报告中的关键词数据和竞品分析撰写
3. 每篇包含：引言、3-5个H2、FAQ、CTA
4. 保存到 `content/posts/[slug].mdx`

### 阶段2：中文本地化（与英文同日完成）
1. 翻译核心内容
2. 替换案例为中国市场可理解版本（如A股/港股/币安）
3. 调整监管提示（CFTC → 中国证监会/外汇管理局等）
4. 保存到 `content/posts/[slug].zh.mdx`

### 阶段3：构建与发布（每日）
```bash
npm run build
```
生成静态文件到 `out/`，部署到 vibetrading.fun。

### 阶段4：GSC提交（每日）
1. 通过GSC URL Inspection手动提交新页面
2. 每日限额约10-20次，优先提交英文页面
3. Day 14批量提交未索引页面

## 七、GSC索引加速动作

1. **每日更新Sitemap**：`npm run build` 自动生成
2. **手动提交**：每篇英文页面发布后立即提交
3. **内链补强**：从新页面链接到已索引的支柱页（如 `best-ai-trading-bots-2026`、`top-ai-trading-github-projects-2026`）
4. **支柱页互链**：确保 `top-ai-trading-github-projects-2026` 链接到所有新的GitHub项目教程
5. **社交/外链信号**：在Reddit r/algotrading、GitHub Discussions、Twitter分享新内容
6. **索引排查**：Day 14检查Coverage报告，修复 "Crawled - currently not indexed" 页面

## 九、内容来源与转化策略

### GitHub转化
- 从 `research/github-ai-trading-projects.md` 提取项目
- 每篇教程包含：安装、最小可运行示例、实际截图/终端输出、风险提示
- 避免纯介绍，必须有"How we tested"段落

### Reddit转化
- 从 `research/reddit-ai-trading-needs.md` 提取痛点
- 每篇以Reddit真实问题开头
- 提供可操作的解决方案和代码/检查清单

### YouTube转化
- 筛选高观看量的AI trading视频主题
- 将视频策略转化为可回测的代码
- 标注原视频来源，补充自己的回测结果

## 十、风险控制

| 风险 | 应对 |
|---|---|
| 内容质量下降 | 严格执行SOP，每篇必含FAQ+Disclaimer+内链 |
| Google不索引 | 每日手动提交+内链补强+检查Coverage |
| 产出跟不上 | 优先保证英文版，中文版可次日补充 |
| 中文监管风险 | 所有中文内容加入"不构成投资建议"声明 |
| 开源项目信息过时 | 每月更新star数和功能描述 |
