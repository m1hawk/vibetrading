财报公告会创造个股中最大且最可预测的一些波动。它们也带来不确定性。交易者常试图猜测涨跌。更好的方法是围绕波动率和财报后漂移而非方向构建策略。

## 为什么财报不同

财报事件把信息发布浓缩到一次公告中，导致：

- 财报前隐含波动率扩张
- 财报后价格大幅跳空
- 事件后隐含波动率崩溃
- 惊喜方向上持续漂移

这些模式创造多个策略角度。

## 策略一：财报前波动率扩张

财报前期权因交易者预期大波动而变贵。一些策略在财报前买入跨式或宽跨式，赌实际波动超过期权定价的预期波动。

简单过滤：

- 财报前 1-3 天买入跨式
- 财报后马上平仓
- 仅当隐含波动率尚未极端时交易

## 策略二：财报后漂移

研究表明，正财报惊喜的股票倾向于在数周内继续走高，反之亦然。AI 模型可预测漂移方向和幅度，使用：

- 财报惊喜 vs 分析师预期
- 业绩指引变化
- 管理层在财报电话会上的语气
- 期权市场隐含波动
- 历史财报后行为

## 财报模型特征工程

```python
df['earnings_surprise'] = (df['actual_eps'] - df['estimated_eps']) / df['estimated_eps']
df['implied_move'] = df['atm_straddle_price'] / df['stock_price']
df['historical_gap'] = df['gap_after_earnings'].rolling(8).mean()
df['sentiment_score'] = df['earnings_call_sentiment']
df['revenue_surprise'] = (df['actual_revenue'] - df['estimated_revenue']) / df['estimated_revenue']
```

## 构建 AI 财报模型

训练分类器预测财报后方向：

```python
from sklearn.ensemble import GradientBoostingClassifier

features = ['earnings_surprise', 'revenue_surprise', 'implied_move', 'historical_gap', 'sentiment_score']
X = df[features]
y = df['next_day_direction']

model = GradientBoostingClassifier()
model.fit(X, y)
```

用预测概率调整仓位。

## 风险管理

财报交易风险高，因为：

- 波动可能远超预期
- 公告前后流动性可能稀薄
- 期权在波动率崩溃后可能迅速贬值
- 指引比 headline 数字更重要

风险规则：

- 每笔交易风险小
- 避免持有过未知事件
- 使用风险有限的期权结构
- 分散多个财报事件

## 数据源

- Yahoo Finance 或 Alpha Vantage 的财报日历
- Polygon 或 Cboe 的期权数据
- 财报电话会记录用于 NLP 分析
- 金融数据提供商的分析师预期

## 总结

AI 财报策略最好利用波动率模式和财报后漂移，而非试图预测精确财报数字。关键是使用大量事件样本、严格控制风险，并避免对任何单一预测过度自信。
